# Events

Events are tuples delivered to your `update/2` callback. The first element
identifies the event family, the rest carries event-specific data.

## Widget events

These are generated by user interaction with widgets. The renderer maps
widget interactions to event tuples using the node's `id`.

### Click

```elixir
{:click, button_id}
```

Generated by `button` widgets when pressed.

```elixir
def update(model, {:click, "save"}), do: save(model)
def update(model, {:click, "cancel"}), do: revert(model)
```

### Input

```elixir
{:input, field_id, value}
```

Generated by `text_input` on every keystroke (when `on_input` is set,
which is the default).

```elixir
def update(model, {:input, "search", value}) do
  %{model | search_query: value}
end
```

### Submit

```elixir
{:submit, field_id, value}
```

Generated by `text_input` when the user presses Enter (when `on_submit`
is set).

```elixir
def update(model, {:submit, "search", query}) do
  {%{model | search_query: query}, search_command(query)}
end
```

### Toggle

```elixir
{:toggle, id, value}
```

Generated by `checkbox` and `toggler`. `value` is `true` or `false`.

```elixir
def update(model, {:toggle, "dark_mode", enabled}) do
  %{model | dark_mode: enabled}
end
```

### Select

```elixir
{:select, id, value}
```

Generated by `pick_list` and `combo_box` when an option is selected.

```elixir
def update(model, {:select, "theme_picker", theme}) do
  %{model | theme: theme}
end
```

### Slide

```elixir
{:slide, id, value}
{:slide_release, id, value}
```

Generated by `slider` and `vertical_slider`. `slide` fires continuously
during dragging. `slide_release` fires once when the user releases the
slider.

```elixir
def update(model, {:slide, "volume", value}) do
  %{model | volume: value}
end

def update(model, {:slide_release, "volume", value}) do
  # Persist the final value
  {%{model | volume: value}, save_preference(:volume, value)}
end
```

### Text editor content change

```elixir
{:input, id, content}
```

Generated by `text_editor` on content changes. The `content` string
contains the full editor text after each edit. This is the same event
family as `text_input`.

```elixir
def update(model, {:input, "notes", content}) do
  %{model | notes_content: content}
end
```

### Key binding

```elixir
{:key_binding, id, data}
```

Generated by `text_editor` widgets when a declarative key binding rule
with a `"custom"` action matches. The `id` is the text editor's node ID.
The `data` is the custom tag string from the binding rule.

```elixir
def update(model, {:key_binding, "editor", "save"}) do
  {model, save_file(model)}
end

def update(model, {:key_binding, "editor", "format"}) do
  %{model | content: format_code(model.content)}
end
```

### Scroll

```elixir
{:scroll, id, %{
  absolute_x: float,
  absolute_y: float,
  relative_x: float,
  relative_y: float,
  bounds: {width, height},
  content_bounds: {width, height}
}}
```

Generated by `scrollable` when the scroll position changes (when
`on_scroll: true` is set). The viewport data includes both absolute pixel
offsets and relative (0.0-1.0) scroll positions, plus the visible bounds
and total content bounds.

```elixir
def update(model, {:scroll, "log_view", viewport}) do
  at_bottom = viewport.relative_y >= 0.99
  %{model | auto_scroll: at_bottom}
end
```

### Paste

```elixir
{:paste, id, text}
```

Generated by `text_input` when the user pastes text (when `on_paste: true`
is set). The `text` field contains the pasted string.

```elixir
def update(model, {:paste, "url_input", text}) do
  %{model | url: String.trim(text)}
end
```

### Option hovered

```elixir
{:option_hovered, id, value}
```

Generated by `combo_box` when the user hovers over an option in the
dropdown (when `on_option_hovered: true` is set). The `value` is the
hovered option string.

```elixir
def update(model, {:option_hovered, "search", value}) do
  %{model | preview: value}
end
```

### Open / Close

```elixir
{:open, id}
{:close, id}
```

Generated by `pick_list` and `combo_box` when the dropdown menu opens or
closes (when `on_open: true` and/or `on_close: true` are set).

```elixir
def update(model, {:open, "country_picker"}) do
  %{model | picker_open: true}
end

def update(model, {:close, "country_picker"}) do
  %{model | picker_open: false}
end
```

### Sort

```elixir
{:sort, id, column_key}
```

Generated by `table` when a sortable column header is clicked. The
`column_key` is the string key from the column descriptor.

```elixir
def update(model, {:sort, "users", column_key}) do
  order = if model.sort_by == column_key, do: flip(model.sort_order), else: "asc"
  %{model | sort_by: column_key, sort_order: order}
end
```

### Mouse area events

Mouse area events fall into two categories: unconditional events that
always fire, and opt-in events enabled by boolean props on the
`mouse_area` widget.

**Always emitted (unconditional):**

```elixir
{:click, id}              # left press
{:click, "id:release"}    # left release
```

These use the standard `{:click, _}` family. The release variant appends
a suffix to the id string.

**Opt-in events (enabled via boolean props):**

```elixir
{:mouse_right_press, id}                # right press (prop: on_right_press: true)
{:mouse_right_release, id}              # right release (prop: on_right_release: true)
{:mouse_middle_press, id}               # middle press (prop: on_middle_press: true)
{:mouse_middle_release, id}             # middle release (prop: on_middle_release: true)
{:mouse_double_click, id}              # double click (prop: on_double_click: true)
{:mouse_enter, id}                      # cursor entered (prop: on_enter: true)
{:mouse_exit, id}                       # cursor left (prop: on_exit: true)
{:mouse_move, id, x, y}                # cursor moved (prop: on_move: true)
{:mouse_scroll, id, delta_x, delta_y}  # scroll wheel (prop: on_scroll: true)
```

Each opt-in event requires its corresponding boolean prop to be set on the
mouse_area widget. Without the prop, the event is not emitted.

```elixir
def update(model, {:mouse_enter, "hover_zone"}) do
  %{model | hovered: true}
end

def update(model, {:mouse_move, "canvas_area", x, y}) do
  %{model | cursor: {x, y}}
end
```

### Canvas events

Generated by `canvas` widgets. Each event is opt-in via a boolean prop on
the canvas node.

```elixir
{:canvas_press, id, x, y, button}
{:canvas_release, id, x, y, button}
{:canvas_move, id, x, y}
{:canvas_scroll, id, x, y, delta_x, delta_y}
```

The `button` field is a string (`"left"`, `"right"`, `"middle"`). The
`x`/`y` coordinates are relative to the canvas origin.

```elixir
def update(model, {:canvas_press, "draw_area", x, y, "left"}) do
  %{model | drawing: true, last_point: {x, y}}
end

def update(model, {:canvas_move, "draw_area", x, y}) do
  if model.drawing do
    %{model | last_point: {x, y}, strokes: [{x, y} | model.strokes]}
  else
    model
  end
end
```

### Sensor events

Generated by `sensor` widgets when the sensor detects a size change.

```elixir
{:sensor_resize, id, width, height}
```

```elixir
def update(model, {:sensor_resize, "content_area", width, height}) do
  %{model | content_size: {width, height}}
end
```

### PaneGrid events

Generated by `pane_grid` widgets during pane interactions.

```elixir
{:pane_resized, id, split, ratio}
{:pane_dragged, id, pane, target}
{:pane_clicked, id, pane}
```

```elixir
def update(model, {:pane_resized, "editor", split, ratio}) do
  put_in(model, [:splits, split], ratio)
end

def update(model, {:pane_clicked, "editor", pane}) do
  %{model | active_pane: pane}
end
```

## Keyboard events

Delivered when keyboard subscriptions are active (see
[commands.md](commands.md)).

```elixir
{:key_press, %Julep.KeyEvent{}}
{:key_release, %Julep.KeyEvent{}}
```

### KeyEvent struct

```elixir
%Julep.KeyEvent{
  key: :enter | "a" | ...,           # logical key (named atom or character string)
  modified_key: :enter | "A" | ...,  # key with modifiers applied (e.g. Shift+a = "A")
  physical_key: atom(),               # physical key code (layout-independent)
  location: :standard | :left | :right | :numpad,
  modifiers: %Julep.KeyModifiers{},
  text: String.t() | nil,            # text produced by the key press (nil for non-printable)
  repeat: boolean()                   # true if this is an auto-repeat event
}
```

### KeyModifiers struct

```elixir
%Julep.KeyModifiers{
  shift: boolean(),
  ctrl: boolean(),
  alt: boolean(),
  logo: boolean(),
  command: boolean()    # platform-aware: ctrl on Linux/Windows, logo (Cmd) on macOS
}
```

### Key values

Named keys are atoms:

```elixir
:enter, :escape, :tab, :backspace, :delete,
:arrow_up, :arrow_down, :arrow_left, :arrow_right,
:home, :end, :page_up, :page_down,
:f1, :f2, ... :f12,
:space
```

Character keys are single-character strings:

```elixir
"a", "b", "1", "/", " "
```

### Keyboard event examples

```elixir
def update(model, {:key_press, %Julep.KeyEvent{key: "s", modifiers: %{command: true}}}) do
  {model, save_command()}
end

def update(model, {:key_press, %Julep.KeyEvent{key: :escape}}) do
  %{model | modal_open: false}
end

def update(model, {:key_press, %Julep.KeyEvent{key: "z", modifiers: %{command: true, shift: false}}}) do
  undo(model)
end

def update(model, {:key_press, %Julep.KeyEvent{key: "z", modifiers: %{command: true, shift: true}}}) do
  redo(model)
end

# Use physical_key for layout-independent bindings (e.g. WASD on non-QWERTY)
def update(model, {:key_press, %Julep.KeyEvent{physical_key: :key_w}}) do
  move_up(model)
end

# Use text field for text input handling
def update(model, {:key_press, %Julep.KeyEvent{text: text}}) when text != nil do
  append_char(model, text)
end
```

## IME events

Delivered when IME subscriptions are active (see
[commands.md](commands.md)). Input Method Editor events support CJK and
other compose-based text input.

```elixir
{:ime_opened}
{:ime_preedit, text, {start, end} | nil}
{:ime_commit, text}
{:ime_closed}
```

`preedit` fires during composition with the in-progress text and optional
cursor range. `commit` fires when the user finalises a composed string.

```elixir
def update(model, {:ime_preedit, text, _cursor}) do
  %{model | composing: text}
end

def update(model, {:ime_commit, text}) do
  %{model | composing: nil, value: model.value <> text}
end
```

## Mouse events (global)

Delivered when mouse subscriptions are active. These are global
(not widget-scoped) events from the windowing system.

```elixir
{:cursor_moved, x, y}
{:cursor_entered}
{:cursor_left}
{:button_pressed, button}
{:button_released, button}
{:wheel_scrolled, delta_x, delta_y, unit}
```

The `button` field is a string (`"left"`, `"right"`, `"middle"`).
The `unit` field indicates scroll units (`"line"` or `"pixel"`).

```elixir
def update(model, {:cursor_moved, x, y}) do
  %{model | cursor: {x, y}}
end

def update(model, {:button_pressed, "left"}) do
  %{model | mouse_down: true}
end
```

## Touch events

Delivered when touch subscriptions are active.

```elixir
{:finger_pressed, finger_id, x, y}
{:finger_moved, finger_id, x, y}
{:finger_lifted, finger_id, x, y}
{:finger_lost, finger_id, x, y}
```

```elixir
def update(model, {:finger_pressed, _finger_id, x, y}) do
  %{model | touch_start: {x, y}}
end
```

## Modifier state events

Delivered when modifier key state changes (subscription-driven).

```elixir
{:modifiers_changed, %Julep.KeyModifiers{}}
```

```elixir
def update(model, {:modifiers_changed, mods}) do
  %{model | shift_held: mods.shift}
end
```

## Window events

Delivered when window subscriptions are active or for lifecycle events
on windows the app manages.

```elixir
{:window_close_requested, window_id}
{:window_opened, window_id, position, {width, height}}
{:window_closed, window_id}
{:window_moved, window_id, x, y}
{:window_resized, window_id, width, height}
{:window_focused, window_id}
{:window_unfocused, window_id}
{:window_rescaled, window_id, scale_factor}
{:file_hovered, window_id, path}
{:file_dropped, window_id, path}
{:files_hovered_left, window_id}
{:theme_changed, mode}
```

The `position` in `window_opened` is `{x, y}` or `nil`. The `mode` in
`theme_changed` is `"light"` or `"dark"`.

### Handling window close

```elixir
def update(model, {:window_close_requested, "main"}) do
  if model.unsaved_changes do
    %{model | confirm_exit: true}
  else
    {model, Julep.Command.close_window("main")}
  end
end
```

### File drag and drop

```elixir
def update(model, {:file_hovered, "main", path}) do
  %{model | drop_target_active: true, hovered_file: path}
end

def update(model, {:file_dropped, "main", path}) do
  {%{model | drop_target_active: false}, load_file(path)}
end

def update(model, {:files_hovered_left, "main"}) do
  %{model | drop_target_active: false}
end
```

## System events

```elixir
{:animation_frame, timestamp}
```

Delivered on each animation frame when an animation subscription is
active. The `timestamp` is the frame time.

## Timer events

Delivered by timer subscriptions.

```elixir
{:tick, timestamp}
```

Where `timestamp` is the monotonic time in milliseconds.

## Command result events

Delivered when an async command completes.

```elixir
{event_tag, result}
```

Where `event_tag` is the atom or tuple you passed to
`Julep.Command.async/2`.

```elixir
def update(model, {:click, "fetch"}) do
  {model, Julep.Command.async(fn -> HTTP.get!("/api/data") end, :data_loaded)}
end

def update(model, {:data_loaded, %{body: body}}) do
  %{model | data: body}
end
```

## Effect result events

Delivered when a renderer effect completes.

```elixir
{:effect_result, id, {:ok, value} | {:error, reason}}
```

The `id` is the auto-generated effect ID (e.g. `"ef_1234"`), not the
effect kind name.

```elixir
def update(model, {:effect_result, _id, {:ok, path}}) do
  {model, load_file(path)}
end

def update(model, {:effect_result, _id, {:error, reason}}) do
  %{model | error: reason}
end
```

See [effects.md](effects.md).

## Catch-all

Always include a catch-all clause:

```elixir
def update(model, _event), do: model
```

Unknown events are silently ignored. This is important for forward
compatibility -- new widget types or renderer versions may emit events
your app does not yet handle.

## Pattern matching tips

Events are plain tuples, so all of Elixir's pattern matching works:

```elixir
# Match any click
def update(model, {:click, id}) do
  IO.inspect(id, label: "clicked")
  handle_click(model, id)
end

# Match clicks with a prefix
def update(model, {:click, "nav:" <> section}) do
  %{model | section: String.to_existing_atom(section)}
end

# Match toggle on any "setting:" prefixed checkbox
def update(model, {:toggle, "setting:" <> key, value}) do
  put_in(model, [:settings, String.to_existing_atom(key)], value)
end
```
